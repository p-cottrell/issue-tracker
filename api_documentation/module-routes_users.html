Module: routes/users

User Management Routes

This module defines routes for handling the following user-related tasks: registration, log-in, log-out, user token validation, retrieval of current user, retrieval of all users, update the current user's username, update the current user's password, and delete a user. Each route uses JWT-based authentication to ensure that tasks can only be carried out authorized users.

IMPORTANT: `authenticateToken` middleware authenticates using cookies, so when calling the API from the front-end, you must use `{ withCredentials: true }` to ensure authentication cookies are passed.
